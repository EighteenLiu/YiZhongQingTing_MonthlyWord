# 可回收物交投点月报生成系统

本项目是一个本地 Web 工具，用于上传 Excel 检查数据和 Word 月报模板，自动提取检查记录和现场图片，统计问题指标，并在模板格式基础上生成 Word 月报 `.docx`。

## 功能

- Flask 本地页面上传 `.xlsx` / `.xls`
- 上传 `.docx` 月报模板，并基于模板写入新月报
- 自动识别街道、点位、点位类型、问题描述、检查日期等字段
- 从转换后的 `.xlsx` 中提取图片，并按图片锚点所在行匹配记录
- 根据问题描述关键词识别问题指标
- 生成总体情况、各街道案例、附件统计表
- 提供 Word 下载链接
- 在页面展示缺失字段、未匹配图片、图片数量异常等日志

## 安装

```bash
cd monthly_report_system
pip install -r requirements.txt
```

## 运行

```bash
python app.py
```

启动后打开：

```text
http://127.0.0.1:8502
```

页面中上传 Excel 检查数据和 Word 模板，填写检查周期、报告月份、输出地址和输出文件名，点击“生成 Word 月报”。

## Word 模板要求

模板必须是 `.docx` 文件。系统会读取模板的样式库、页眉页脚和页面设置，然后清空正文并写入新月报内容。

这样生成的月报会尽量遵循模板总体格式，同时正文仍按系统要求生成：

- 标题
- 一、总体情况
- 二、各街道案例
- 点位图片
- 附件统计表

## Excel 表格要求

默认读取第一张工作表。表格中建议包含以下字段：

- 街道名称
- 点位名称
- 点位类型
- 检查结果
- 问题描述
- 检查日期
- 现场照片

字段名不必完全一致，系统会做别名映射。例如：

- `街道`、`街道名称`、`所属街道` 会识别为街道
- `点位`、`点位名称`、`地址` 会识别为点位
- `问题`、`问题描述`、`检查问题` 会识别为问题描述
- `图片`、`现场照片`、`整改前照片`、`检查照片` 会识别为图片字段

图片需要直接嵌入 Excel 工作表中。系统会按图片左上角锚点所在行与数据记录关联。每条记录期望 3 张图片；如果数量不是 3，页面会显示提示。

## xls 文件处理

`openpyxl` 不能直接读取 `.xls`，也不能直接解析 `.xls` 中的嵌入图片。本系统会先把 `.xls` 转换为 `.xlsx`，再使用转换后的 `.xlsx` 进行后续读取、图片提取、统计和 Word 生成。

1. 上传 `.xls`
2. 程序明确调用 Microsoft Excel 的 `Excel.Application` COM 对象
3. 不调用 WPS，也不遵从电脑的默认打开方式
4. 转换成功后，系统读取转换后的 `.xlsx`
5. 后续图片提取也基于转换后的 `.xlsx`

为了尽量保证图片无损，系统不会用 pandas/openpyxl 重写 `.xls` 文件，而是使用 Excel 自身的 `SaveAs` 完成格式升级，保留嵌入图片对象、锚点和工作表布局。即使电脑上 `.xls` 默认打开方式是 WPS，也不会影响本程序的转换选择。若转换失败，请确认 Microsoft Excel 已安装并正确注册 COM 组件，或手动将 `.xls` 另存为 `.xlsx` 后上传。

## 问题指标识别

当前支持以下附件统计指标：

- 无防火标识
- 无称重系统
- 无回收价格表
- 无灭火器
- 灭火器未成组
- 灭火器不合格
- 占道经营
- 无消防安全水源
- 周边环境脏乱
- 无七禁收八不准承诺书

如果 Excel 中没有独立指标列，系统会从“问题描述”中按关键词识别。例如“缺失防火标识”会计入“无防火标识”，“灭火器不合格”会计入“灭火器不合格”。

## 输出文件

页面可填写输出地址。默认保存到：

```text
monthly_report_system/output/
```

临时文件、转换后的 `.xlsx` 和提取图片保存到：

```text
monthly_report_system/temp/
```

## 常见问题

### 读取不到图片

请确认图片是嵌入 Excel 的真实图片对象，而不是单元格里的链接或文本路径。若图片锚点离数据行较远，可能无法匹配到对应记录。

### 上传 xls 后转换失败

请确认本机安装了 Microsoft Excel 或 WPS，并允许自动化调用。也可以手动将 `.xls` 另存为 `.xlsx` 后再上传。

### 生成 Word 时提示文件被占用

请关闭已经打开的同名 Word 文件后重试，或换一个输出文件名。

### 统计指标不准确

可以在 `config/indicators.py` 中维护 `INDICATOR_KEYWORDS`，增加本单位常用表述。
