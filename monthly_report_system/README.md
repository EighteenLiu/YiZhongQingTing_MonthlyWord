# 可回收物交投点月报生成系统

本项目是一个本地桌面工具，用于选择 Excel 检查数据和 Word 月报模板，自动提取记录、图片和问题指标，并生成 `.docx` 月报。

## 运行

双击或执行：

```bat
run_app.bat
```

首次运行会在项目目录下创建 `.venv` 虚拟环境并安装依赖；后续会直接复用该环境启动桌面窗口。

## 使用流程

1. 选择 Excel 检查数据文件，支持 `.xls` / `.xlsx`。
2. 选择 Word 月报模板，支持 `.docx`。
3. 填写报告标题、检查日期、报告月份、输出目录和输出文件名。
4. 点击“生成 Word 月报”。
5. 生成完成后可在窗口中查看统计、预览、日志，并点击“打开文件”查看 Word。

## Excel 读取规则

- `.xls` 会先通过 Microsoft Excel COM 原生转换为 `.xlsx`，不会调用 WPS，也不会跟随系统默认打开方式。
- `.xlsx` 直接读取第一张工作表。
- 支持两行表头：当第二行包含 `3级点位`、`4级点位` 等字段时，系统使用第二行作为业务表头，并从第三行开始读取数据。
- `3级点位` 会作为街道名和二级标题来源。
- `4级点位` 会作为点位名和三级标题来源。
- `具体问题` 会作为点位标题后的正文内容。
- 图片按 Excel 图片锚点所在行匹配，每条记录最多保留 3 张。

## Word 生成规则

- 基于上传的 `.docx` 模板生成，保留模板样式、页眉页脚和页面设置。
- 一级标题：`一、总体情况`、`二、各街道案例`、附件标题。
- 二级标题示例：`（一）西长安街街道`，使用 Word `Heading 2`。
- 三级标题示例：`1.广州大厦可回收物交投点：`，使用 Word `Heading 3`。
- 三级标题后写入 `具体问题` 内容，然后直接横向插入最多三张图片。
- 图片插入尺寸为 `5.25cm x 3.06cm`，不放入表格。

## 输出与临时文件

- 生成的 Word 默认保存到 `monthly_report_system/output/`。
- 处理过程中的转换文件和压缩图片保存到 `monthly_report_system/temp/run_*`。
- 旧的 `run_*` 临时目录会按保留周期自动清理。
